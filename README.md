# ts-design-pattern

## creational Pattern

- [ ] factory method
  - 객체를 생성하는 factory
- [ ] abstract factory
  - factory의 interface정의 후 여러 factory를 운영
- [ ] builder
  - constructor에 Long Parameter List 문제가 발생하는 복잡한 객체의 경우 단계적으로 객체를 생성하는 패턴
  - director class is optional
- [ ] object cloning
  - 객체 clone은 해당 객체의 책임. 객체의 비공개 메서드 등 외부에서는 완전한 clone이 불가하며 비효율적
